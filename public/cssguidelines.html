CSS Guidelines ▲

General Guidelines

  • Write style rules in CSS files. Avoid writing embedded, or inline CSS.
  • Load app-specific style sheet last in the stack.
  • The :hover selector should not be used as :hover does not apply to touchscreens.
  • Use flex properties, where applicable, to dynamically size content to available space.
  • Use relative units, where possible, over absolute units in property values.

Z-indexing

Z-index is a property that allows for stacking elements in the direction of the user. The primary use in UI design for z-index is to display overlapping elements.

  • Only use to overlay elements on top of others on the z-axis.
  • Only use when the document flow order cannot be changed.
  • Infinite or arbitrary z-indexing should not be used. (eg. z-index: 9999)
  • Start with a base value of "1" and move up in increments of 1 as needed.
  • Z-indexing should be combined with absolute positioning (absolute, fixed) on one of the elements within the group of overlapping elements.

Class and ID Naming

Excerpted from Google HTML/CSS Style Guide > ID and Class Name Delimiters

Separate words in ID and class names by a hyphen. Do not concatenate words and abbreviations in selectors by any characters (including none at all) other than hyphens, in order to improve understanding and scannability.

/* Not recommended: does not separate the words “demo” and “image” */
.demoimage {}

/* Not recommended: uses underscore instead of hyphen */
.error_status {}

/* Not recommended: uses camelCase instead of hyphen, hard to read */
.hexSliderButton {}
 
/* Recommended */
#video-id {}
.ads-sample {}

CSS Formatting

Excerpted from Google HTML/CSS Style Guide > CSS Formatting Rulles

  • Use only lowercase 
    /* Not recommended */
color: #E5E5E5;
     
    /* Recommended */
color: #e5e5e5;
  • Alphabetize declarations 
    background: fuchsia;
border: 1px solid;
-moz-border-radius: 4px;
-webkit-border-radius: 4px;
border-radius: 4px;
color: black;
text-align: center;
text-indent: 2em;
  • Indent all block content 
    html {
background: #fff;
color: #444;
}
     
    html {
  background: #fff;
  color: #444;
}
  • Use a semicolon after every declaration 
    /* Not recommended */
.test {
  display: block;
  height: 100px
}
     
    /* Recommended */
.test {
  display: block;
  height: 100px;
}
  • Use a space after a property name’s colon 
    /* Not recommended */
h3 {
  font-weight:bold;
}
     
    /* Recommended */
h3 {
  font-weight: bold;
}
  • Use a space between the last selector and the declaration block 
    /* Not recommended: missing space */
#video{
  margin-top: 1em;
}

/* Not recommended: unnecessary line break */
#video
{
  margin-top: 1em;
}
     
    /* Recommended */
#video {
  margin-top: 1em;
}
  • Separate selectors and declarations by new lines, and
  • Always start a new line for each selector and declaration 
    /* Not recommended */
a:focus, a:active {
  position: relative; top: 1px;
}
     
    /* Recommended */
h1,
h2,
h3 {
  font-weight: normal;
  line-height: 1.2;
}
  • Separate rules by new lines 
    html {
  background: #fff;
}

body {
  margin: auto;
  width: 50%;
}
