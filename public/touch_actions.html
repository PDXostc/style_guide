touch-action is a CSS property that specifies whether and how touch input can be received by the user. The default value for touch-action is auto. The remaining values are none, pan-x, pan-y, and manipulation.

The issue for DNA:

All elements inherit the touch-action value of auto from the user-agent. Therefore, when an element that has scrolling or zooming behavior (eg. an overflowing list) is touched, all the DOM elements receive the touch behavior. This results in unwanted panning and zooming of the entire application.

The fix for DNA:

The touch-action property has been set to none across the app via the body element. All the DOM elements will now inherit this value and not have the ability to be scrolled or zoomed. The side-affect is that it disables scrolling of overflowed elements.

To regain scrolling, the touch-action property needs to be reset at the level of the scroll element. The following style rules have been added to common.css to assist in quickly enabling scrolling on any element.
