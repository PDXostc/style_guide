Scrolling Content

Touch Action

The touch-action CSS property specifies whether and how a given region can be manipulated by the user (for instance, by panning or zooming).1

The touch-action property has been set to none across the app suite via the body element. This was implemented to prevent unwanted zooming and scrolling of the viewport area. The side-effect is that it disables the ability to scroll overflow content.

The ul element (unordered list) and the ol element (ordered list) have been updated to allow for scrolling.

To enable scrolling on any other element, use any of the following classes:

Class Description
.scrollable-x Allows horizontal scrolling without a scrollbar
.scrollable-y Allows vertical scrolling without a scrollbar
.scrollable-xy Allows horizontal and vertical scrolling without a scrollbar
.scrollbar-x Allows horizontal scrolling with a scrollbar
.scrollbar-y Allows vertical scrolling with a scrollbar
.scrollbar-xy Allows horizontal and vertical scrolling with a scrollbar
