Color Guidelines

Color Breakdown
  • The primary colors are orange and blue.
  • The secondary colors are yellow and green.
  • The neutral colors are grey, white, and black.
Color Usage
  • Primary and secondary colors are provided in light and vivid versions.
  • The standard color is the light color.
  • Vivid colors are used for highlighting, emphasis, button and form states, or where needed for contrast.
Color Values
  • Hexadecimal values are preferred.
  • RGB values can be used for incorporating alpha values via RGBA.
Neutral Usage
  • Black is used primarily as a background color or text on light backgrounds.
  • White and light grey are used as borders and text.

Sliders

Sliders may implement a black bar and heagonal peg, round peg or square peg for position and control

Volume

Volume control shall be done with black background that will be filled with a colour to illustrate level of volume along the bar and single hexagonal peg.

Scroll bar

Scroll bars should be a solid line of white, black or grey, otherwise one of the colors from the color palette should be used.

To my knowledge, browser scrollbars can't be styled with CSS, only with some tricky JS. - MM

CSS Guidelines ▲

This section is work-in-progress.

All styling should be written in CSS files. Avoid writing embedded, or inline CSS. Javascript should be as unobtrusive as possible and use styles and IDs to modify the DOM rather than writing attribute and CSS property values.

Viewport Settings
  • Total area is 1080 x 1920 (w/h) which should scale down to 720x1280 proportionately.
Stylesheet Links
  • The app specific style sheet shall be loaded last in the stack.
Writing CSS
  • Inline or embedding CSS should be avoided.
  • Default styles should not be overwritten.
  • The :hover selector should not be used as :hover does not apply to touchscreens.
Containers
  • Flex properties should be used where applicable.
Units
  • Relative units are preferred over absolute units in property values.
Z-indexing
  • Z-index should start as a base value of "1" and move up in single incremented up by 1 as needed for each overlaying elements.
  • Infinite or arbitrary z-indexing should not be used. (eg. z-index: 9999)
  • Z-index should only be used when the stack order cannot be changed.
  • Z-index should only be used on two or more elements. Z-index on one element has no effect.
Inputs
  • The flashing input caret should be used on :focus elements that accept keyboard events or other user inputs.

HTML Guidelines ▲

This section is work-in-progress. Content here may be broken out to other sections.

  • HTML pages should not contain blocks of JavaScript code using the script tag.
  • The script tag shall only be used for attaching external pieces of JavaScript code.
  • Elements should not contain event attributes (eg. onclick="someFunction")
